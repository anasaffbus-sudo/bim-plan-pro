1

ما هو ISO 19650؟

ISO 19650 هو المعيار الدولي لإدارة المعلومات على مدى دورة حياة الأصل المبني باستخدام نمذجة معلومات البناء (BIM). يوفر إطاراً موحداً لتنظيم وتبادل المعلومات بين جميع الأطراف المشاركة في المشروع.

الهدف الرئيسي: ضمان إنتاج المعلومات الصحيحة، في الوقت المناسب، بالكمية الكافية، وبالجودة المطلوبة لاتخاذ قرارات فعالة خلال دورة حياة المشروع.

2

أجزاء معيار ISO 19650

الجزء 1

ISO 19650-1

المفاهيم والمبادئ الأساسية لإدارة المعلومات باستخدام BIM

الجزء 2

ISO 19650-2

مرحلة تسليم الأصول - كيفية إدارة المعلومات أثناء التصميم والبناء

الجزء 3

ISO 19650-3

مرحلة تشغيل الأصول - إدارة المعلومات أثناء التشغيل والصيانة

الجزء 5

ISO 19650-5

النهج الأمني لإدارة المعلومات - حماية البيانات الحساسة

3

المفاهيم الأساسية

متطلبات المعلومات (Information Requirements)

  • OIR - متطلبات المعلومات التنظيمية: ما تحتاجه المنظمة لتحقيق أهدافها
  • PIR - متطلبات معلومات المشروع: المعلومات اللازمة لاتخاذ قرارات المشروع
  • AIR - متطلبات معلومات الأصل: المعلومات المطلوبة لإدارة الأصل
  • EIR - متطلبات تبادل المعلومات: ما يجب على كل طرف تقديمه

مستوى المعلومات المطلوب (Level of Information Need)

يشمل ثلاثة أبعاد لكل عنصر في النموذج:

  • التفاصيل الهندسية: مستوى دقة الشكل ثلاثي الأبعاد
  • البيانات الوصفية: الخصائص والمعلومات المرفقة بالعنصر
  • التوثيق: الوثائق والمستندات المرتبطة

خطط تسليم المعلومات

  • MIDP - خطة تسليم المعلومات الرئيسية: الجدول الشامل لجميع التسليمات
  • TIDP - خطة تسليم المعلومات الخاصة بالفريق: التزامات كل فريق
  • BEP - خطة تنفيذ BIM: الوثيقة الشاملة لعملية إدارة المعلومات
4

الأدوار والمسؤوليات

الدور بالإنجليزية الوصف
الطرف المعيّن Appointing Party المالك أو العميل الذي يحدد متطلبات المعلومات ويقبل التسليمات
الطرف المعيّن الرئيسي Lead Appointed Party الجهة الرئيسية المسؤولة عن تنسيق عمل جميع الفرق وتجميع النماذج
الأطراف المعيّنة Appointed Parties الفرق المتخصصة (معماري، إنشائي، MEP) التي تنتج النماذج
مدير المعلومات Information Manager المسؤول عن إدارة CDE ومراقبة جودة المعلومات والالتزام بالمعايير

بيئة البيانات المشتركة (CDE)

بيئة البيانات المشتركة هي المصدر الوحيد للمعلومات في المشروع. تتكون من أربع حالات:

WIP

SHARED

المشترك

مشاركة المعلومات بين الفرق للتنسيق والمراجعة

PUBLISHED

ARCHIVED

6

خطوات كتابة خطة تنفيذ BIM (BEP)

1

تحديد معلومات المشروع

توثيق بيانات المشروع الأساسية: الاسم، الموقع، النوع، الحجم، الأطراف المعنية، والجدول الزمني.

2

تحديد الأدوار والمسؤوليات

تعيين الطرف المعيّن والطرف الرئيسي ومدير المعلومات وفرق المهام مع تحديد مسؤوليات كل طرف.

3

إعداد استراتيجية تسليم المعلومات

تحديد مستويات المعلومات المطلوبة لكل مرحلة، ومواعيد التسليم، ومعايير القبول.

4

إعداد بروتوكول CDE

تحديد نظام CDE، هيكل المجلدات، صلاحيات الوصول، وسير عمل المراجعة والاعتماد.

5

تحديد المعايير التقنية

البرامج المعتمدة، معيار التسمية، نظام التصنيف، صيغ التبادل، نظام الإحداثيات والوحدات.

6

وضع خطة ضمان الجودة

إجراءات الفحص الداخلي والخارجي، قوائم المراجعة، ومعايير القبول لكل تسليم.

7

إدارة المخاطر والأمن

تحديد المخاطر المحتملة، إجراءات التخفيف، وسياسة أمن المعلومات.

7

أفضل الممارسات

الاتفاق المبكر

اتفق على معايير BIM وخطة التنفيذ مبكراً قبل بدء أعمال التصميم

التحديث المستمر

حدّث خطة BEP بشكل دوري مع تطور المشروع وتغير المتطلبات

التدريب

تأكد من تدريب جميع الفرق على المعايير والأدوات المعتمدة

فحص الجودة

طبّق فحوصات الجودة بشكل منتظم ولا تتأخر في حل المشاكل

التواصل الفعال

عقد اجتماعات تنسيق BIM منتظمة وتوثيق القرارات

أمن المعلومات

طبّق سياسات أمنية صارمة وراقب صلاحيات الوصول

8

التحليل المقارن والفجوات (Comparative Gap Analysis)

هذا القسم يرفع الدليل من مستوى الملخص السريع إلى مستوى أقرب للتدقيق المهني، عبر مقارنة مباشرة مع متطلبات ISO 19650-1 وISO 19650-2 وISO 19650-3 وISO 19650-5.

ملاحظة استراتيجية: نجاح التطبيق لا يعتمد فقط على الإجراءات التنفيذية، بل على تضمين المنظورات الأربع لإدارة المعلومات: منظور المالك، المستخدم، المنفذ، والمجتمع (Clause 4.3).

نقاط مغطاة بشكل جيد

  • هيكلة متطلبات المعلومات OIR/PIR/AIR/EIR.
  • تبني مفهوم Level of Information Need بأبعاده الثلاثة.
  • استخدام MIDP وTIDP كآلية تخطيط للتسليم.

فجوات تحتاج تعميق

  • تقييم القدرة والسعة (Capability and Capacity) قبل التعيين.
  • الانتقال من مصفوفة أدوار عامة إلى مصفوفة مسؤوليات تفصيلية لكل حاوية معلومات.
  • ربط AIR بأحداث تشغيلية فعلية بدل اعتبارها قائمة ثابتة.
9

عمليات ISO 19650-2 التفصيلية لمرحلة التسليم

في المشاريع الكبيرة، يُوصى بالعمل وفق دورة العمليات الكاملة لمرحلة التسليم، وليس فقط خطوات عامة. الترتيب التالي يضمن وضوح التعاقد والتسليمات والتحقق:

  1. تقييم الحاجة وتحديد متطلبات المعلومات على مستوى التعيين.
  2. الدعوة للتعاقد (Invitation to tender) مع تجميع واضح لمتطلبات المعلومات ضمن وثائق المناقصة.
  3. الردود الفنية من الأطراف المعينة وإثبات القدرة والسعة.
  4. التعيين الرسمي وتأكيد مسؤوليات Lead Appointed Party وAppointed Parties.
  5. إعداد وتطوير BEP قبل التنفيذ التفصيلي (Pre-appointment/Post-appointment).
  6. بناء MIDP/TIDP مع مصفوفة مسؤوليات مفصلة تربط كل مخرج معلومات بمالك واضح.
  7. دورات الإنتاج والفحص الذاتي والتحقق قبل النشر داخل CDE.
  8. اعتماد التسليمات وقبولها رسميًا وتغذية سجل التعلم للمراحل التالية.
أثر إغفال مرحلة المناقصة: يزيد احتمالية سوء تسعير الجهد المعلوماتي، وضعف جودة التزامات الفرق، وظهور خلافات تعاقدية لاحقًا حول نطاق البيانات المطلوبة.
10

مرحلة التشغيل والأمن: ISO 19650-3 وISO 19650-5

القيمة التشغيلية الحقيقية تظهر بعد التسليم. لذلك يجب أن يكون AIM ديناميكيًا ويتم تحديثه عند أحداث تشغيلية محفزة (Trigger Events) وليس فقط في نقاط تسليم المشروع.

أمثلة Trigger Events

  • صيانة طارئة لنظام حرج.
  • استبدال أصل رئيسي أو مكون عالي الكلفة.
  • تعديل تشغيلي بسبب متطلبات تنظيمية أو سلامة.
  • حادث أمني أو انقطاع تشغيلي يستلزم تحديث البيانات.

Sensitivity Triage (ISO 19650-5)

  • تصنيف الأصل حسب الحساسية (مثل ST1 إلى ST4).
  • تحديد ضوابط الوصول، المشاركة، والتخزين لكل مستوى.
  • تعريف إجراءات تصعيد أمنية للأصول الحساسة.
  • ربط قرارات الأمن بالمخاطر الفعلية وليس بسياسة عامة فقط.
تحذير مهني: في الأصول الحساسة (مطارات، سجون، مراكز بيانات، بنية حرجة)، لا يكفي بند "أمن المعلومات" العام. يجب تطبيق منهجية Triage موثقة وقابلة للتدقيق.
11

حوكمة CDE والبيانات الوصفية (Metadata Governance)

الـ CDE ليس مجرد شجرة مجلدات، بل نظام إدارة بيانات وصفية. لتحقيق أتمتة مراجعة واعتماد فعالة، يجب تعريف السمات الإلزامية لكل حاوية معلومات.

السمة الإلزامية الغرض أثر غيابها
Revision Code تتبع النسخ والتغييرات عبر الزمن. فقدان اليقين حول آخر نسخة معتمدة.
Status Code تحديد حالة الاستخدام (مراجعة/تنسيق/اعتماد/تنفيذ). التباس في صلاحية استخدام الملف للقرار أو التنفيذ.
Container Owner تحديد الطرف المسؤول عن الجودة والتحديث. تضارب المسؤوليات وتأخر معالجة الملاحظات.
Approval Record إثبات مسار المراجعة والاعتماد. ضعف قابلية التدقيق والامتثال التعاقدي.
12

جاهزية التدقيق والتوصيات التنفيذية

استخدم القائمة التالية كأداة فحص قبل أي تدقيق داخلي أو خارجي لمطابقة ISO 19650:

  • تم توثيق متطلبات المعلومات وتحديثها دوريًا (OIR/PIR/AIR/EIR).
  • تم إجراء تقييم Capability & Capacity قبل التعيين.
  • مصفوفة المسؤوليات تربط كل عنصر معلومات بجهة واضحة.
  • مراحل CDE مرتبطة بحالات حالة/مراجعة موحدة وقابلة للتتبع.
  • Trigger Events معرفة وتؤدي لتحديث AIM بشكل رسمي.
  • تم تطبيق Sensitivity Triage للأصول الحساسة مع ضوابط موثقة.

التوصية العملية: استخدم هذا الدليل كمرجع تشغيلي يومي، لكن في المشاريع الحساسة أو عالية المخاطر يجب الرجوع المباشر للنصوص الأصلية، خصوصًا ISO 19650-5، مع توثيق قرارات الأمن والمسؤوليات بشكل قابل للتدقيق.
